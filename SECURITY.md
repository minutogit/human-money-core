# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

We take the security of `human-money-core` very seriously. If you discover a security vulnerability, please do **not** open a public issue.

### Responsible Disclosure

Please report vulnerabilities by emailing **[INSERT SECURITY EMAIL]** or by reaching out to the maintainers directly. Ideally, use PGP to encrypt your communication.

We will acknowledge your report within 48 hours and provide an estimated timeframe for a fix. We ask that you give us a reasonable amount of time to fix the vulnerability before disclosing it to the public.

### Security Updates

Security updates will be released as patch versions. We recommend all users to keep their dependencies up to date.
